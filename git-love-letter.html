While git init creates a hidden .git file in the folder you initialize in,

nothing can hide when I type ls -la.

And instead of changing our relationship status, I'd always prefer to add a file to the cue by typing git status.


Then, when I'm ready to go,

git add [filename] will let me track a file,

or I could just type git add . to add EVERYTHING in a folder to staging.

Although I'm afraid of commitment,

I'll git commit -m "message" to you any day.

(as long as you don't forget to git add . before comitting)


We have a whole lot of history that I'll never forget,

especially thanks to git log.


If you ever need someone to talk to, just sign into my server using

git config -global user.name "Tyler Schad", as well as git config -global user.email "tysc1075@colorado.edu".

And remember, if reading a git log is too long, just press "q" to exit.
